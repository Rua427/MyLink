## 1. 프로젝트 개요
- **프로젝트명:** 마이링크 (MyLink)
- **목적:** 개발자 및 크리에이터를 위한 여러 개의 링크를 하나의 통합된 페이지로 모아서 공유할 수 있게 해주는 마이크로 랜딩 페이지 서비스 (Linktree 클론)
- **대상 사용자:** 
  - 다양한 소셜 미디어(유튜브, 인스타그램 등) 채널을 운영하는 크리에이터
  - 깃허브(GitHub), 기술 블로그(Velog, Medium 등), 포트폴리오를 한 곳에 모아 공유하고 싶은 개발자 및 디자이너

## 2. 핵심 기능 목록

### 필수 기능 (MVP - Minimum Viable Product)
- **회원가입 및 로그인:** Firebase를 활용한 구글 소셜 로그인 전용
- **프로필 관리 (인라인 편집):** 표시 이름(displayName), 한 줄 소개(Bio) 등록 및 수정 (이미지 업로드 없음)
- **링크 관리 (CRUD 및 인라인 편집):**
  - 링크 추가, 수정, 삭제
  - 링크 수정 시 별도의 모달이나 페이지 이동 없는 인라인 모드(Inline Editing) 적용
  - 링크 정렬: 최신순(가장 최근에 추가한 링크가 상단에 위치) 정렬 고정
  - 링크 아이템 기본 아이콘: 구글 API를 활용한 URL 파비콘(Favicon) 자동 추출 및 표시
- **개인화된 페이지 제공:** `mylink.com/username` 형태의 고유 URL 할당 및 제공
- **UI 시스템:** shadcn/ui 기반의 직관적인 디자인 (별도 커스텀 테마 기능 없음)

## 3. 기능 상세 설명

### 3.1 회원가입 및 로그인
- **Firebase Authentication의 구글 로그인**을 사용하여 사용자가 단 1번의 클릭으로 가입 및 로그인할 수 있도록 합니다.
- 최초 로그인 시 고유한 `username`을 설정해야 하며, 이는 개인 페이지의 고유 식별자(URL)로 사용됩니다. 중복 검사는 전용 최적화 컬렉션을 통해 실시간으로 처리됩니다.

### 3.2 프로필 관리
- **이미지 없음:** 프로필 이미지 업로드 기능은 제공하지 않습니다. (텍스트 위주의 깔끔한 구성)
- **인라인 편집 (Inline Editing):** 
  - **표시 이름 (displayName):** 페이지 상단에 표시될 닉네임입니다. 텍스트를 클릭하여 바로 수정할 수 있는 인라인 편집 모드를 지원합니다.
  - **한 줄 소개 (Bio):** 방문자에게 자신을 소개할 수 있는 텍스트 필드입니다. 이 역시 텍스트 클릭 시 즉시 편집 가능한 인라인 모드를 지원합니다.

### 3.3 링크 관리
- **추가/삭제:** 직관적인 UI를 통해 링크 블록을 새롭게 추가하거나 삭제할 수 있습니다.
- **인라인 편집 (Inline Editing):** 생성된 링크의 타이틀이나 URL을 수정할 때, 텍스트 부분을 클릭하면 곧바로 수정 상태로 변경되는 인라인 편집을 지원하여 사용자 경험(UX)을 향상시킵니다.
- **정렬 방식:** 별도의 순서 변경 기능 없이, 링크 생성일자(`createdAt`)를 기준으로 내림차순(최신순)으로 자동 정렬됩니다.
- **파비콘(Favicon) 연동:** 구글 파비콘 추출 API(`https://www.google.com/s2/favicons?domain=...`)를 활용하여 해당 웹사이트의 파비콘을 빠르고 안정적으로 가져옵니다.
- **유효성 검사:** 입력된 URL 형식이 올바른지 검사합니다 (http/https 포함 여부).

### 3.4 UI 및 디자인 시스템 (shadcn/ui)
- 대시보드(관리자 페이지)와 사용자 노출 페이지 모두 **shadcn/ui** 기반의 컴포넌트를 활용하여 통일감 있고 깔끔한 디자인을 구현합니다.
- 복잡한 테마 변경 기능이나 배경 커스터마이징 기능은 배제합니다.

## 4. 데이터베이스 모델링 (Firebase Firestore)

조회 속도 및 읽기 비용 최적화를 위해 `username` 중복 검사용 전용 컬렉션을 운영하며, 개별 링크 데이터는 서브 컬렉션(`links`)으로 구성합니다.

### 4.1 `usernames` 컬렉션 (중복 검사 최적화용)
- **Document ID:** 사용자가 설정한 `username` 텍스트 자체
- **Fields:**
  - `uid` (string): 해당 유저네임을 소유한 사용자의 UID (Auth UID)
- **설명:** 회원가입 시 문서 ID 조회를 통해 O(1)의 속도로 빠르게 중복 여부를 파악합니다.

### 4.2 `users` 메인 컬렉션
- **Document ID:** 사용자의 고유 UID (Firebase Auth UID)
- **Fields:**
  - `username` (string): 개인 페이지 URL에 사용되는 고유 ID
  - `displayName` (string): 화면 상단에 표시되는 사용자 닉네임
  - `bio` (string): 한 줄 소개
  - `createdAt` (timestamp): 계정 생성일자
  - `updatedAt` (timestamp): 계정 마지막 수정일자
- *(참고: 이미지 업로드 기능 제외로 `profileImageUrl` 필드 제거됨)*

### 4.3 `links` 서브 컬렉션 (`users/{uid}/links`)
- **Document ID:** 링크 블록의 고유 ID (자동 생성)
- **Fields:**
  - `title` (string): 링크 제목
  - `url` (string): 연결될 URL 주소
  - `faviconUrl` (string): 구글 API를 통해 추출한 파비콘 이미지 URL 경로
  - `createdAt` (timestamp): 링크 생성일자 (목록 정렬 시 활용 기준, 최신순)
  - `updatedAt` (timestamp): 링크 마지막 수정일자

## 5. 기술 스택
- **프론트엔드:** Next.js (React), Tailwind CSS, shadcn/ui
- **백엔드/인증/데이터베이스:** Firebase (Authentication, Firestore)
